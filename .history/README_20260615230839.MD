## Troubleshooting

- Styles not loading: confirm the stylesheet path in `index.html` matches `css/style.css` and that the file is present.
- Fonts not loading: open the browser devtools network tab to verify `assets/fonts/` requests succeed and that `@font-face` paths are correct.
- Images missing: ensure files in `assets/images/` (including `screenshot.png`) exist and the paths in HTML/CSS are correct.
- Live site shows old content after deploy: clear CDN cache or trigger a re-deploy from your hosting dashboard.
- Keyboard focus not visible: make sure `outline` or focus styles are not globally removed (e.g., `:focus { outline: none; }`).
- Local server 404s static files: run the server from the project root so relative paths resolve correctly.

If a problem persists, open an issue in the repository and include a short description, browser and OS, and steps to reproduce.

## Accessibility notes

- Navigation and social links include ARIA labels where appropriate.
- Focus styles are visible for keyboard users.
- Colors chosen to meet contrast recommendations — verify with a contrast checker when customizing colors.

Example focus style (from `style.css`):

```css
.social-links a:focus {
  background-color: hsl(75, 94%, 57%);
  color: hsl(0, 0%, 8%);
  outline: 3px solid hsl(75, 94%, 57%);
  outline-offset: 2px;
}
```
